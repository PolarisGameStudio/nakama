### 2.5 **CRITICAL BUG found + fixed during QA · `bracket_tournament_*` "function invalid"** 🛑→✅

Booted upstream `heroiclabs/nakama:3.38.0` in Docker with the PR bundle mounted. All 7 `bracket_tournament_*` RPCs returned:
```
JavaScript runtime function invalid. key:__rpc_bracket_tournament_<name>
```
…while everything else (Hiro, Satori, QuizVerse, MpKernel, library.countdown, legacy) worked.

**Root cause:** `BracketTournaments.register(initializer: Initializer, logger: Logger)` was the only module that combined:
1. **String-LITERAL** `initializer.registerRpc("bracket_tournament_create", rpcCreate)` (→ postbuild rewrites these into `__rpc_bracket_tournament_create = rpcCreate` global assignments at IIFE scope and injects a late `registerRpc("bracket_tournament_create", __rpc_bracket_tournament_create)` replay in the new InitModule wrapper); **and**
2. **Multi-arg signature** (→ postbuild skips IIFE auto-invoke for multi-arg `register()` functions; the skip is even printed in the postbuild log).

Result: the `__rpc_bracket_tournament_*` globals stayed `undefined` in every pooled Goja VM (pooled VMs re-load the bundle but do **not** re-run InitModule), so the late replay registered `undefined` as the handler. Every call → "function invalid".

QuizVerse / LibraryCountdown / MpKernel evade this because they use **string CONSTANTS** as the first arg of `registerRpc` (`QuizVersePlugin.RPC_CREATE_MATCH`, etc.), which the postbuild does *not* rewrite — so the registerRpc fires inline inside `main.ts` → InitModule and the AST walker resolves the handler to a module-scope function reference that pooled VMs can find.

**Fix (commit `b90fbdd8` on PR #54):**
* `BracketTournaments.register(initializer, logger)` → `register(initializer)` (single-arg)
* The `logger.info("[BracketTournaments] RPCs registered (7)")` moved from inside the function body to the call site in `main.ts`
* Added a top-of-function comment so the next person who adds a `register()` doesn't re-introduce the footgun
* Bundle regenerated — postbuild output went from `Auto-invoked register() at IIFE scope (86 build…)` to `(87 build…)`.
